Pyspark:
	It is the python API for apache spark a powerful open source 
	"Distibuted computing system" for big data processing.
	Pyspark support both "batch processing and stream processing".
	
	Introduction:
		- Master : Manage the spark jobs
		- Driver : Initiate the spark session
		- executor(Node Manager) : processing the task with it's memory and cores
		- worker : Like data node to process the data with executor.
		- Dag : Design the execution pipe line based on the action.[optimized execution plan]
		- Lineage : to balance the spark core from the fault tolerance, Have it's dependencies. 
			[rrd3 from where its created rrd1][Data handle by lineage]
		- lazy evalution : execeute the transformation once after call the action.[to save the process time]
		- Stages : Initiate the session to follow the narrow/Wide Transformations. 
			each transformation create a new stage. It run on the Dag backend as a abstract class.
		- Transformation : Narrow(Non shuffled) , Wide (shuffled)
		- Action : Pipe line execution.
		
	Flow:
	- spark.py 
	- driver program -> initiate the spark context
	- DAG -> Plan the best way of execution(DAG) -> Task scheduler [multiple task scheduler] -> Runs on the Node manager.
	- YARN -> allocate the resource as per the job -> Allocated job ready to be in Node Manager(executors).
	
	Shared variable:
		- Shared Variables are a feature of the programming language APL which allows 
		  APL programs running on one processor to share information with another processor.
			- APL programming:
				- Unlike traditionally structured programming languages, APL code is typically 
					structured as chains of monadic or dyadic functions, and operators acting on arrays.
		- Types:
			- Broadcast variable:
				- used to cache a value in memory on all nodes
			- Accumulators:
				- which are variables that are only “added” to, such as counters and sums.
	
		
	RDD: (Resilient Distribution Dataset)
		- It is Immutable.
			Why:
			 - simplicity : Once RDD created, it cannot be changed. It makes easy to understand and work with data.
			 - parallel Processing : process across multiple machine. perform operation on different parts of data simultaneously.
			 - fault tolerance : can recover from failures, or recreate the RDD using "linear graph".
			 - Cache and Optimization : enable fast processing because of immutable.

			RDD creation:
			1. Using parallelize():
				dataList = [("Java", 20000), ("Python", 100000), ("Scala", 3000)]
				rdd=spark.sparkContext.parallelize(dataList)
			2. Using textFile():
				rdd2 = spark.sparkContext.textFile("/path/test.txt")
			3. create RDD from existing one:
				While creating transformation it get creating new RDD.
				
	DataFrame
		- A DataFrame is a distributed collection of data organized into named columns. It is a more powerful abstraction than RDDs, 
			and it provides a richer set of APIs for data manipulation.
		- DataFrames can be manipulated using a variety of SQL-like operations [spark.sql import SparkSession]. 
			They can also be used to create machine learning models.
		- df = createOrReplaceTempView("view") -> ss.sql("access data from view")
		
		DataFrames can be created from a variety of sources, including:
		- Structured data files , Tables in Hive
		- External databases , Existing RDDs
	
	Dataset
		- A Dataset is a new abstraction in Spark that combines the benefits of RDDs and DataFrames. 
		- It is a distributed collection of data that is both immutable and strongly typed. 
		- This makes it a good choice for both batch and streaming applications.
		- Datasets can be created from a variety of sources, just like DataFrames. 
		- They can also be manipulated using a variety of SQL-like operations. 
		- However, Datasets also provide the benefits of strong typing, which can help to improve performance and correctness.

Spark Batch process:

	1. Import PySpark SQL:
	   from pyspark.sql import SparkSession

	2. Create a SparkSession:
	   spark = SparkSession.builder \
		   .appName("PySpark SQL Tutorial") \
		   .getOrCreate()
	3. Load Data:
	   - You can load data from various sources like CSV, JSON, Parquet, etc. For example, to load a CSV file:
		 df = spark.read.csv("path/to/file.csv", header=True, inferSchema=True)
	4. Explore Data:
	   - You can check the schema and preview the data using the following methods:
		 df.printSchema()  # Print the schema of the DataFrame
		 df.show()        # Show the first few rows of the DataFrame
	5. Perform SQL Operations:
	   - PySpark SQL allows you to run SQL-like queries on DataFrames. You can register a DataFrame as a temporary table and then query it using SQL syntax. For example:
		 df.createOrReplaceTempView("my_table")  # Register DataFrame as a temporary table

		 result = spark.sql("SELECT * FROM my_table WHERE age > 30")  # SQL query

		 result.show()  # Show the result
	6. Perform DataFrame Operations:
	   - PySpark SQL provides a rich set of functions to manipulate and transform DataFrames. You can perform various operations like filtering, aggregating, joining, etc. For example:
		 result = df.filter(df.age > 30)  # Filter records based on a condition

		 result = df.groupBy("gender").count()  # Group by a column and perform an aggregation

		 result = df.join(another_df, "id", "inner")  # Join two DataFrames on a common column

		 result.show()
	7. Write Data:
	   - You can write the processed data back to a file or a database. For example, to write a DataFrame to a Parquet file:
		 result.write.parquet("path/to/output.parquet")
	
Pyspark Streaming process:

	1. Import PySpark Streaming:
	   from pyspark.streaming import StreamingContext
	2. Create a StreamingContext:
	   - A StreamingContext is the entry point for streaming functionality in PySpark. It requires two parameters: SparkContext and the batch interval (duration) in which the streaming data will be divided.
	   from pyspark import SparkContext

	   sc = SparkContext(appName="PySpark Streaming Tutorial")
	   ssc = StreamingContext(sparkContext=sc, batchDuration=1)  # Batch interval of 1 second
	3. Define Input DStream:
	   - DStream (Discretized Stream) represents a continuous stream of data. You can create DStreams from various sources like Kafka, Flume, HDFS, etc. For example, to create a DStream from a text file directory:
	   dstream = ssc.textFileStream("path/to/directory")
	4. Apply Transformations:
	   - PySpark Streaming supports many transformations that can be applied to DStreams. Some commonly used transformations include `map`, `filter`, `reduceByKey`, etc. For example, to count the occurrences of each word in the stream:
	   wordCounts = dstream.flatMap(lambda line: line.split(" ")) \
						   .map(lambda word: (word, 1)) \
						   .reduceByKey(lambda a, b: a + b)
	5. Define Output Operations:
	   - Output operations allow you to specify what you want to do with the processed data. Common output operations include `print`, `saveAsTextFiles`, `foreachRDD`, etc. For example, to print the word counts:
	   wordCounts.pprint()
	6. Start the StreamingContext:
	   - After defining the DStreams and output operations, you can start the streaming process using the `start()` method of the StreamingContext.
	   ssc.start()
	7. Await Termination:
	   - To keep the streaming application running, you can use the `awaitTermination()` method. This will allow the application to run until it is manually stopped.
	   ssc.awaitTermination()
	8. Gracefully Stop the StreamingContext:
	   - To stop the streaming context, you can use the `stop()` method. It is important to gracefully stop the context to ensure that all the resources are released properly.
	   ssc.stop()
