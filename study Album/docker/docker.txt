Docker :

1. intro to containers:
	Defination:
		- It is also virtualization(alternate for hypervisor) technique.
		- any application can bundled no worries about dependencies. It is isolated environment
		- It containerized Environment.
		
	- developer devlope a code in py version 2 and when it moved to UAT it will be not working due to the version
		because the UAT team try to execute the code in a py version 3.
	- Here containers help to solve this kind of problems.
	- BUILD => SHIP => RUN
	
	Environment:
		1. Infrastructure
		2. Operating system
		3. Docker (dockerd)
		4. App [With bundled environment] [like py -n venv {env_name}]
	
	Advantage:
		- Containers are isolated. Incase of any issue in app other app with same name will keep on running the app.
		- Portability, Light weight, Fast Delivery and scalable
		- Docker used for continuous deployment and Testing.
		- Ability to run multiple isolated OS on single host.
		- Resorce optimization.
		
	Disadvantage:
		- If host operating system get down Then all app will be crashed.

4. Docker Images:
	Defination:
		- It is a environment that has a all nessessery file for running the application.
		- It is a template of a Docker container. Very lightweight. single images can create different environment (Dev, Prod, UAT)
		- It consist of many layers with unique image ID from base images.
		- It is read only layer od docker, containers are read write layer of docker image.
	- Without docker image we cannot run anything.
	- can get images form "dockerhub" website 
		=> [sudo] docker pull {httpd}
	- sudo docker images
	- [sudo] docker inspect [doc_name]
	- sudo docker run -d -it --name {app_name} -p 8001:8081 {image_name}
	- sudo docker ps -a
	- sudo docker save {con_name| con_id} > backup.tar
	- pull docker images from offline
		scp backup.tar thamu@192.168.2.13:/home/thamu/
	- load offline images into docker
		sudo docker load -i backup.tar
	- remove images:
		sudo docker stop {image_name}
		sudo docker rm {image_name|id} {con_name|id}
		sudo docker rmi {image_name|id} => for remove all : sudo docker rmi ${sudo "docker images -a -q"}
	- after changing the change into docker image save:
		sudo docker commit {con_name|id} {forsetting_Name_for image}
	- push image to docker hub
		sudo docker login => sign up into docker hub use the same id.
		=> sudo docker tag {customimgae:2.1} {userid/customimgae:2.1} (It will change the name of the image name)
		sudo docker push {userid/customimgae:2.1}
		
5. Docker File:
	Defination:
		- Defining the list of instruction or configuration
	Instruction:
		=> save file as normal textfile with named {dockerfile}
		=> sudo docker build .
		- Everything is starts from FROM
			- FROM:
				FROM docker.io/{os-> centos:latest}
				
			- MAINTAINER:
				- MAINTAINER Admistrator
				- MAINTAINER admin @ {somthing.net}
				- MAINTAINER devopes Engg(admin @ somthing.net)
				
			- LABEL:
				- LABEL {app_env} = {"devlopement"}
				- LABEL {"app_support"} = {"learn.net group"}
				
			- EXPOSE: (used for setting the listening port)
				- EXPOSE 80 443
				- EXPOSE 80/tcp 8080/udp
				
			- ADD (used for compressed file copy) || - COPY
				- ADD /root/* /data/	
				
				- COPY src dest
			
			- RUN: used for running command
				- RUN sudo update
				- RUN systemctl start httpd
				- RUN ["sudo", "update"]
			
			- CMD: executable command comes under the CMD
				- CMD ping google.com && CMD apachectl -D FOREGROUND
				- ENTRYPOINT["ping", "google.com"] && CMD ["python", "app.py"]
			
			- VOLUME : for mount the excess volume
				- VOLUME /data

			- USER:
				- USER webadmin
				- USER 2489749
				
			- WORKDIR:
				- WORKDIR /app/
				
			- ENV: 
				- ENV wrkdir /data/ (permanent)
				- ARG temp mycustom_image (temp)

Docker Network:
	- by default the network has been start on 
		- docker run ubuntu (it will run on bridge = 172.17.0.1 [another containers 2, 3 ..])
	- to change:
		- docker run --network=none ubuntu
		- docker run --network=host ubuntu
		
	- to check list of networks
		- sudo docker network ls [--help]
		- sudo apt-get install net-tools, telnet, apt-get update
		- sudo docker run -d -it --name {container_name} --network={host|none ubuntu} 
		- sudo docker run -d -it --name {container_name} -P httpd
	
Docker Volume:
	- the Volume is more important because if we are deleting the containers no longer the data will be available.
	- So we need to be allocated some Volume it help to keep our data.
	
	Types:
		- Bind mount -> data stored into -> docker area
		- volume -> data stored into -> docker area
		- tmpfs mount -> Memory
		
	- sudo docker volume --help
	- sudo docker volume ls
	- sudo docker volume create {vol_name}
	- sudo docker volume inspect {vol_name}
	- sudo docker run -d -it --name {con_name} --mount source={vol_name},destination=/data /bin/bash
